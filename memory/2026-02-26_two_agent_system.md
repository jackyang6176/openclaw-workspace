# 🦞 雙人驗證系統正式實施 - 2026-02-26

## 用戶指令

**用戶明確要求**：雙人驗證機制不僅用於投資分析，必須應用於**以後所有指派的工作任務**。

## 系統配置

### Agent 配置

| 角色 | Agent ID | Model | Workspace | 職責 |
|------|----------|-------|-----------|------|
| **DOER** | `main` | `qwen3.5-plus` | `/home/admin/.openclaw/workspace` | 執行任務、生成內容、產出初稿 |
| **VERIFIER** | `deepseek` | `qwen3-max-thinking` | `/home/admin/.openclaw/workspace-deepseek` | 驗證所有輸出、檢查數據準確性、標記錯誤 |

### 強制驗證範圍

**所有用戶指派的工作任務都必須通過 VERIFIER 驗證**，包括但不限於：

#### 投資與財務
- 投資分析與交易建議
- 財務監控報告
- 信用卡/繳費通知
- 技術分析

#### 資訊與報告
- 新聞聚合與市場數據
- 工作報告與摘要
- 數據分析與統計

#### 技術與開發
- 代碼生成與審查
- 系統配置變更
- 數據庫操作
- 安全敏感操作

#### 日常任務
- 郵件與通訊草稿
- 日程與提醒設置
- 文檔編寫與編輯
- 計劃與策略制定

#### 技能學習與擴展
- 新技能安裝與配置
- 知識整理與總結
- 測試與驗證結果

## 驗證流程

```
用戶指派任務
    ↓
DOER (main + qwen3.5-plus) 生成初稿
    ↓
VERIFIER (deepseek + qwen3-max-thinking) 驗證
    ↓
【APPROVED】→ 發送給用戶
【REJECTED】→ 退回 DOER 修正 → 重新驗證
【APPROVED WITH CONDITIONS】→ DOER 處理條件 → 發送
```

## 驗證輸出格式

VERIFIER 必須使用以下格式輸出驗證報告：

```markdown
## 驗證報告

### ✓ 確認正確項目
- [列出驗證通過的項目]

### ⚠️ 發現問題
- [列出問題或遺漏]

### ✗ 錯誤或可疑內容
- [列出錯誤或需要進一步驗證的內容]

### 💡 改進建議
- [具體改進建議]

### 數據可信度評分：XX/100

### 最終裁決：【APPROVED】或【REJECTED】或【APPROVED WITH CONDITIONS】
```

## 質量門檻

- ✅ **所有用戶指派任務**必須通過 VERIFIER 才能交付
- ✅ 任何來自未驗證來源的數據必須被標記並拒絕
- ✅ 虛構/模板化內容必須【REJECTED】並退回 DOER
- ✅ 槓桿產品必須包含風險警告
- ✅ DOER 必須在用戶看到輸出前修正**所有**錯誤

## 迭代限制

- 最多 **3 次**修正循環
- 如果 3 次循環後仍不能【APPROVED】，則附上完整錯誤報告升級給用戶
