# 專案管理技能 (Project Management Skill)

## 📋 技能描述

協助用戶進行專案規劃、執行、監控和收尾的完整技能。

## 🎯 使用場景

使用此技能當：
- 用戶需要規劃新專案
- 需要追蹤專案進度
- 需要風險管理建議
- 需要專案模板或檢查清單
- 需要敏捷/Scrum 指導
- 需要專案狀態報告

## 💡 功能列表

### 1. 專案啟動
- 創建專案章程
- 識別利害關係人
- 定義專案目標 (SMART 原則)
- 初步風險評估

### 2. 專案規劃
- 工作分解結構 (WBS)
- 時程規劃 (甘特圖)
- 資源分配
- 風險管理計劃
- 溝通計劃

### 3. 專案執行
- 任務分配
- 團隊協作指導
- 變更管理
- 品質保證

### 4. 專案監控
- 進度追蹤
- 成本監控
- 風險監控
- 狀態報告生成

### 5. 專案收尾
- 驗收檢查清單
- 經驗教訓總結
- 文件歸檔
- 團隊解散計劃

## 🔧 命令使用

### 啟動新專案
```
請幫我啟動一個新專案：[專案名稱]
目標：[專案目標]
預計完成：[日期]
```

```
請為 [專案名稱] 創建工作分解結構

```

### 風險評估
```
請幫我進行風險評估

主要關注：[關注點]
```

### 狀態報告
```
生成本週專案狀態報告
專案：[名稱]
本週完成：[列表]
下週計劃：[列表]
```

### 敏捷指導
```
請指導我們如何實施 Scrum
團隊規模：[人數]
專案類型：[類型]
```

## 🎓 專案管理方法論

### 傳統 (Waterfall)
適用：需求明確、變更少的專案

```
需求 → 設計 → 開發 → 測試 → 上線
```

### 敏捷 (Agile)
適用：需求不明確、快速變化的專案

```
Sprint 1 (2-4 週) → 可交付增量
Sprint 2 (2-4 週) → 可交付增量
...
```

### Scrum 框架
```
Product Owner → Product Backlog
     ↓
Scrum Team → Sprint Planning
     ↓
Daily Standup (15 分鐘)
     ↓
Sprint Review + Retrospective
```

## 🚨 常見陷阱與解決方案

### 1. 範圍蔓延 (Scope Creep)
**問題**: 未經控制的變更

**解決方案**:
- 建立變更控制流程
- 所有變更需評估影響
- 利害關係人批准後執行

### 2. 溝通不足
**問題**: 資訊不同步

**解決方案**:
- 定期狀態報告
- 使用統一溝通平台
- 記錄重要決策

### 3. 時程過度樂觀
**問題**: 不切實際的截止日期

**解決方案**:
- 使用三點估算 (樂觀/悲觀/最可能)
- 加入風險緩衝
- 參考歷史數據

### 4. 風險管理不足
**問題**: 被動應對問題

**解決方案**:
- 早期風險識別
- 定期風險檢視
- 準備應變計劃

## 📈 成功指標

### 專案成功標準
- **時程**: 按時完成 (±10%)
- **成本**: 預算內 (±10%)
- **範圍**: 交付所有承諾功能
- **品質**: 缺陷率低於目標
- **滿意度**: 利害關係人滿意度 > 80%

### 團隊健康指標
- 團隊成員留存率
- 團隊滿意度調查
- 衝突解決效率
- 知識分享頻率
